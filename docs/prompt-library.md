# Prompt Library

Copy-paste prompts for common tasks. Customise the `[bracketed]` sections for your needs.

**All prompts follow the CLEAR structure:**
- **C**ontext → **L**anguage → **E**xamples → **A**udience → **R**equest

---

## Summarisation

### Email Chain Summary
```
Context: I have a long email thread about [topic] that I need to brief my manager on quickly.

Language: Concise, factual, action-oriented.

Examples: Structure as: Issue → Key positions → Decision needed.

Audience: My manager, who has no context and 1 minute to read.

Request: Summarise in under 100 words:
- The core issue or request
- Key positions from each party
- What action is needed and by when
```

### Meeting Notes Summary
```
Context: I've just come out of a [meeting type] meeting. I need to share outcomes with people who weren't there.

Language: Neutral, factual. No editorialising.

Examples: Use this structure:
1. Decisions made
2. Action items (owner + deadline)
3. Open questions

Audience: Team members who missed the meeting. They need to know what affects them.

Request: Extract from this transcript:
- All decisions (as a numbered list)
- All action items (with owners and dates if mentioned)
- Open questions or parking lot items
```

### Document Summary
```
Summarise this [document type] for [audience].
Focus on:
- Main argument or purpose
- Key findings or recommendations
- Implications for [our team/department/project]
Length: [X] bullet points or [X] words maximum.
```

### Executive Brief
```
Create an executive brief from this content.
Structure:
- Bottom line up front (1-2 sentences)
- Key points (3-5 bullets)
- Recommended action
- Risks if we don't act
Total length: half a page maximum.
```

## Drafting Communications

### Professional Email
```
Draft an email to [recipient/audience] about [topic].
Tone: [professional/friendly/urgent/diplomatic]
Goal: [what should they do after reading?]
Include:
- [Key point 1]
- [Key point 2]
Length: [short = 3-4 sentences / medium = 2 paragraphs]
```

### Announcement to Team
```
Draft a team announcement about [topic/change].
The team will want to know:
- What is changing
- Why we're making this change
- How it affects them specifically
- When it takes effect
- Who to contact with questions
Tone: [transparent/positive/empathetic]
```

### Sensitive Message
```
Draft a message to [recipient] about [sensitive topic].
Context: [brief background]
I need to convey: [key points]
I want to avoid: [potential misinterpretations]
Tone: direct but [empathetic/respectful/constructive]
Keep it under [X] sentences.
```

### Stakeholder Update
```
Draft a project update for [stakeholder group].
They care most about: [priorities]
Include:
- Progress since last update
- Current status (on track / at risk / blocked)
- Key milestones ahead
- Any decisions or support needed from them
Tone: [confident/candid/measured]
```

## Analysis & Decision Support

### Options Analysis
```
I'm deciding between [Option A] and [Option B].
Context: [situation/constraints]
For each option, provide:
- Top 3 advantages
- Top 3 disadvantages
- Key risks
- Who/what is impacted
Format as a comparison table.
```

### Pros and Cons
```
Give me pros and cons for [decision/approach].
Consider:
- Short-term vs long-term impact
- Cost and resource implications
- Risk factors
- Stakeholder reactions
Format: two columns, bullet points, with a summary recommendation.
```

### Devil's Advocate
```
I'm leaning toward [decision/approach].
Play devil's advocate:
- What are the strongest arguments against this?
- What could go wrong?
- What am I potentially not seeing?
- What would a critic say?
Be direct and specific, not generic.
```

### Risk Assessment
```
For [initiative/project/decision], identify:
1. Top 5 risks (likelihood: high/medium/low)
2. Potential impact of each
3. Suggested mitigation strategies
4. Early warning signs to watch for
Format as a risk register table.
```

## Editing & Improvement

### Make It Shorter
```
Rewrite this to be 50% shorter.
Keep: [key points that must stay]
Cut: [anything unnecessary, redundant, or overly detailed]
Maintain the same [tone/message/call to action].
```

### Change the Tone
```
Rewrite this message.
Current tone: [aggressive/passive/unclear/defensive]
Desired tone: [confident/diplomatic/direct/warm]
Keep the core message the same.
Show me what specific phrases you changed and why.
```

### Make It Clearer
```
This message is confusing. Rewrite it to be clearer.
The main point is: [X]
The action I want: [Y]
Assume the reader has [no/some/deep] context.
Use simple language, short sentences.
```

### Strengthen the Message
```
This is too weak/vague. Strengthen it.
Current: [paste text]
Make it:
- More specific (replace generalities with examples)
- More confident (remove hedging language)
- More compelling (lead with impact)
```

## Function-Specific

### Sales: Proposal Review
```
Review this proposal excerpt.
Identify:
- Where our value proposition is unclear or weak
- Gaps a procurement team might question
- How we could better differentiate from competitors
- Specific language improvements
Audience: [buyer type]
```

### Operations: Process Improvement
```
Analyse this process description.
Identify:
- Inefficiencies or bottlenecks
- Steps that could be automated
- Quick wins (< 1 week to implement)
- Longer-term improvements
Prioritise by impact and effort.
```

### HR: Job Description Review
```
Review this job description for [role].
Evaluate:
- Is it inclusive (language, requirements)?
- Are requirements realistic or inflated?
- What might discourage good candidates?
- How does it compare to market standard?
Suggest specific improvements.
```

### Finance: Explain for Non-Finance
```
Explain this financial information to a non-finance stakeholder.
Avoid: technical jargon, acronyms without explanation
Include: what changed, why it matters, what action is needed
Analogies are helpful.
Keep it under 200 words.
```

## Preparation & Planning

### Meeting Prep
```
I have a meeting with [person/group] about [topic].
Context: [situation]
Prepare me:
- 3-5 questions I should ask them
- Likely concerns or objections they'll raise
- Key points I should make sure to cover
- Potential landmines to avoid
```

### Presentation Outline
```
Create an outline for a [X]-minute presentation on [topic].
Audience: [who]
Goal: [inform/persuade/decide/align]
Key message: [one sentence summary of what I want them to remember]
Include suggested time allocation per section.
```

### Feedback Script
```
I need to give feedback to [person] about [issue].
Use the SBI model:
- Situation: [when/where this happened]
- Behaviour: [what specifically they did]
- Impact: [effect it had]
Tone: direct but [supportive/constructive/developmental]
Include: what I want them to do differently going forward.
```

### Difficult Conversation Prep
```
I need to have a difficult conversation about [topic].
The other person's likely perspective: [X]
My key points: [Y]
Desired outcome: [Z]
Draft:
- An opening that sets the right tone
- Key talking points in order
- How to respond to likely pushback
- A way to close constructively
```

## Formatting Guide

Use these modifiers to control output format:

| What You Want | Add This |
|--------------|----------|
| Bullet points | "Format as bullet points" |
| Numbered list | "Format as a numbered list" |
| Table | "Format as a table with columns for [X], [Y], [Z]" |
| Short | "Maximum [X] words" or "Keep it under [X] bullets" |
| Structured | "Use headings for each section" |
| Prose | "Write in paragraph form, not bullets" |

---

## Tips for Using This Library

1. **Don't use prompts verbatim** — Adapt them to your specific situation
2. **Fill in all brackets** — Missing context = generic output
3. **Iterate** — First attempt rarely perfect; refine based on output
4. **Save your best versions** — Build your own library over time
