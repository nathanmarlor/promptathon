# Prompt Library

Copy-paste prompts for common tasks. Customise the `[bracketed]` sections for your needs.

**All prompts follow the CLEAR structure:**
- **C**ontext → **L**anguage → **E**xamples → **A**udience → **R**equest

---

## Summarisation

### Email Chain Summary
```
Context: I have a long email thread about [topic] that I need to brief my manager on quickly.

Language: Concise, factual, action-oriented.

Examples: Structure as: Issue → Key positions → Decision needed.

Audience: My manager, who has no context and 1 minute to read.

Request: Summarise in under 100 words:
- The core issue or request
- Key positions from each party
- What action is needed and by when
```

### Meeting Notes Summary
```
Context: I've just come out of a [meeting type] meeting. I need to share outcomes with people who weren't there.

Language: Neutral, factual. No editorialising.

Examples: Use this structure:
1. Decisions made
2. Action items (owner + deadline)
3. Open questions

Audience: Team members who missed the meeting. They need to know what affects them.

Request: Extract from this transcript:
- All decisions (as a numbered list)
- All action items (with owners and dates if mentioned)
- Open questions or parking lot items
```

### Document Summary
```
Context: I have a [document type] that I need to understand quickly before [situation].

Language: Plain English, avoiding jargon. Focus on implications.

Examples: Structure as:
- Main argument (1 sentence)
- Key findings (3-5 bullets)
- What this means for us

Audience: [Role/team] who need to know how this affects [their work/project].

Request: Summarise in [X] bullet points or [X] words maximum. Focus on implications for [our team/department/project].
```

### Executive Brief
```
Context: I need to present this content to senior leadership who have limited time.

Language: Direct, confident, no hedging. Bottom line up front.

Examples: Structure as:
- BLUF (1-2 sentences)
- Key points (3-5 bullets)
- Recommended action
- Risks if we don't act

Audience: Executives who will skim and want the "so what" immediately.

Request: Create a half-page executive brief. Lead with the recommendation, then support it.
```

## Drafting Communications

### Professional Email
```
Context: I need to email [recipient/audience] about [topic]. This is [urgent/routine/sensitive].

Language: [Professional/friendly/urgent/diplomatic]. Avoid [specific phrases to exclude].

Examples: Structure as:
- Opening: [what to lead with]
- Body: [key points]
- Close: [call to action]

Audience: [Recipient] who [what they know/care about].

Request: Draft a [short/medium] email. The reader should [specific action] after reading.
```

### Announcement to Team
```
Context: We're announcing [change/news] to the team. This affects [who/what].

Language: Transparent, [positive/empathetic/matter-of-fact]. No corporate-speak.

Examples: Structure as:
1. What is changing
2. Why we're making this change
3. How it affects them specifically
4. When it takes effect
5. Who to contact with questions

Audience: [Team/department] who will want to know how this impacts their work.

Request: Draft the announcement. Anticipate and address the top 3 questions they'll have.
```

### Sensitive Message
```
Context: I need to communicate with [recipient] about [sensitive topic]. Background: [brief context].

Language: Direct but [empathetic/respectful/constructive]. Avoid [potential triggers].

Examples: Keep it tight:
- Opening: acknowledge the situation
- Body: the key message
- Close: next steps or offer of support

Audience: [Recipient] who may react [emotionally/defensively/with concern].

Request: Draft a message under [X] sentences. Convey [key points] while avoiding [potential misinterpretations].
```

### Stakeholder Update
```
Context: I'm providing a [weekly/monthly] update on [project/initiative] to [stakeholder group].

Language: [Confident/candid/measured]. Data-driven where possible.

Examples: Structure as:
- Progress since last update
- Current status (on track / at risk / blocked)
- Key milestones ahead
- Decisions or support needed

Audience: [Stakeholder group] who care most about [their priorities].

Request: Draft an update that highlights what matters to them. Flag any risks or asks clearly.
```

## Analysis & Decision Support

### Options Analysis
```
Context: I'm deciding between [Option A] and [Option B] for [situation]. Constraints: [budget/time/resources].

Language: Analytical, balanced. Present trade-offs fairly without bias.

Examples: Structure as a comparison table with:
- Advantages (3 per option)
- Disadvantages (3 per option)
- Key risks
- Impact on [stakeholders/timeline/budget]

Audience: [Decision-makers] who need a clear recommendation.

Request: Analyse both options. End with your recommendation and the key deciding factor.
```

### Devil's Advocate
```
Context: I'm leaning toward [decision/approach]. I want to stress-test this before committing.

Language: Direct and challenging. Don't soften the critique.

Examples: Structure as:
1. Strongest arguments against
2. What could go wrong (specific scenarios)
3. What I might be missing
4. What a critic would say

Audience: Me — I need honest pushback, not validation.

Request: Play devil's advocate. Be specific to my situation, not generic. What would make me reconsider?
```

### Risk Assessment
```
Context: We're planning [initiative/project/decision]. I need to identify what could go wrong.

Language: Structured, comprehensive. Use standard risk categories.

Examples: Format as a risk register table:
| Risk | Likelihood | Impact | Mitigation | Owner |

Audience: [Project governance/leadership] who need to understand and approve risk exposure.

Request: Identify 5-7 key risks. For each: likelihood (H/M/L), impact, mitigation strategy, and early warning signs.
```

### Strategic Recommendation
```
Context: I need to recommend [action/approach] to [audience]. The alternatives are [X, Y, Z].

Language: Confident, evidence-based. Acknowledge trade-offs honestly.

Examples: Structure as:
- Recommendation (1 sentence)
- Why this option (3 reasons)
- What we're trading off
- How to mitigate downsides
- Implementation next steps

Audience: [Decision-makers] who will challenge and approve.

Request: Build the case for [preferred option]. Address the obvious objections proactively.
```

## Editing & Improvement

### Make It Shorter
```
Context: This message is too long. I need to cut it by 50% without losing the core message.

Language: Same tone as original. Tighter, no redundancy.

Examples: 
- Cut: filler words, redundant phrases, unnecessary context
- Keep: [key points that must stay]

Audience: Same as the original.

Request: Rewrite at half the length. Maintain the [tone/message/call to action]. Show me what you cut.
```

### Change the Tone
```
Context: This message sounds [aggressive/passive/unclear/defensive]. I need to fix it.

Language: Change from [current tone] to [desired tone: confident/diplomatic/direct/warm].

Examples: 
- Replace hedging ("I think maybe...") with direct statements
- Replace aggressive phrases with firm but respectful ones
- Keep the core message identical

Audience: [Same recipient as original].

Request: Rewrite with the new tone. Show me what specific phrases you changed and why.
```

### Make It Clearer
```
Context: I've written this but it's confusing. The reader keeps misunderstanding.

Language: Simple, direct. Short sentences. No jargon.

Examples: 
- One idea per sentence
- Active voice
- Concrete examples instead of abstractions

Audience: Reader who has [no/some/deep] context on this topic.

Request: Rewrite for clarity. The main point is [X]. The action I want is [Y]. Make both unmistakable.
```

### Strengthen the Message
```
Context: This is too weak/vague. It's not landing with the audience.

Language: Confident, specific. No hedging words.

Examples:
- Replace "I think" with "I recommend"
- Replace generalities with specific examples
- Lead with impact, not process

Audience: [Recipient] who needs to take [action].

Request: Strengthen this message. Make it more specific, confident, and compelling. Lead with the impact.
```

## Function-Specific

### Sales: Proposal Review
```
Context: We're sending this proposal to [customer type]. Deal size: [X]. Competitive situation: [context].

Language: Critical but constructive. Flag issues clearly.

Examples: Structure feedback as:
1. Strengths to keep
2. Gaps to address
3. Specific rewrites

Audience: Me and my team — we need to strengthen this before sending.

Request: Review and identify:
- Where our value proposition is unclear
- Gaps procurement might question
- How to differentiate from [competitor]
- Specific language improvements
```

### Operations: Process Improvement
```
Context: Here's a description of our [process name] process. We're looking to improve efficiency.

Language: Analytical, practical. Focus on actionable improvements.

Examples: Categorise findings as:
- Quick wins (< 1 week)
- Medium effort (1-4 weeks)
- Strategic (requires investment)

Audience: Operations leadership evaluating improvement options.

Request: Analyse and identify:
- Bottlenecks or inefficiencies
- Automation opportunities
- Prioritised recommendations by impact and effort
```

### HR: Job Description Review
```
Context: We're hiring for [role]. Here's the draft JD. Market conditions: [competitive/niche].

Language: Critical, inclusive-aware. Flag issues diplomatically.

Examples: Structure as:
- What works
- What to change
- Suggested rewrites

Audience: Hiring manager and HR partner.

Request: Evaluate:
- Inclusive language
- Realistic vs inflated requirements
- What might deter good candidates
- Comparison to market standard
```

### Finance: Explain for Non-Finance
```
Context: I need to explain this [variance/result/metric] to non-finance stakeholders.

Language: Plain English. No jargon unless defined. Lead with business impact.

Examples: Structure as:
- What changed (1 sentence)
- Why it matters (business impact)
- What we should do

Audience: [Business leaders] who understand the business but not accounting mechanics.

Request: Explain in under 200 words. Use an analogy if helpful. Focus on "so what," not methodology.
```

## Preparation & Planning

### Meeting Prep
```
Context: I have a meeting with [person/group] about [topic]. Background: [situation].

Language: Strategic, anticipatory. Help me prepare.

Examples: Structure as:
- Questions to ask
- Points to make
- Concerns to anticipate
- Landmines to avoid

Audience: Me — preparing for this specific meeting.

Request: Prepare me with:
- 3-5 questions I should ask them
- Their likely concerns or objections
- Key points I must cover
- Potential landmines to avoid
```

### Presentation Outline
```
Context: I'm presenting on [topic] for [X] minutes. Goal: [inform/persuade/decide/align].

Language: [Inspiring/analytical/urgent] depending on goal.

Examples: Structure as:
- Opening hook
- 3-4 main sections
- Closing call to action

Audience: [Who] — they care about [what].

Request: Create an outline with:
- Suggested time per section
- Key message for each section
- One thing I want them to remember
```

### Feedback Script
```
Context: I need to give feedback to [person] about [issue]. Relationship: [direct report/peer/senior].

Language: Direct but [supportive/constructive/developmental]. Use SBI model.

Examples: Structure as:
- Situation (when/where)
- Behaviour (what specifically they did)
- Impact (effect it had)
- Request (what to do differently)

Audience: Me — delivering this feedback in person.

Request: Draft a script including how to open, the feedback itself, and how to close constructively.
```

### Difficult Conversation Prep
```
Context: I need to have a difficult conversation about [topic]. Their likely perspective: [X].

Language: Calm, firm, empathetic. Not confrontational.

Examples: Structure as:
- Opening that sets the right tone
- Key talking points in order
- Responses to likely pushback
- Constructive close

Audience: [Person] who may react [defensively/emotionally/with pushback].

Request: Prepare me with:
- How to open
- What to say (in order)
- How to handle their likely responses
- How to close constructively
```

## Quick Modifiers

Add these to any prompt to control format:

| What You Want | Add This |
|--------------|----------|
| Bullet points | "Format as bullet points" |
| Numbered list | "Format as a numbered list" |
| Table | "Format as a table with columns for [X], [Y], [Z]" |
| Short | "Maximum [X] words" or "Keep it under [X] bullets" |
| Structured | "Use headings for each section" |
| Prose | "Write in paragraph form, not bullets" |

## Tips for Using This Library

1. **Don't use prompts verbatim** — Adapt them to your specific situation
2. **Fill in all brackets** — Missing context = generic output
3. **Iterate** — First attempt rarely perfect; refine based on output
4. **Save your best versions** — Build your own library over time
